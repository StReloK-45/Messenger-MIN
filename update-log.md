# История обновлений (Changelog)

## [1.2.1] - 2026-05-15

### Крупное обновление: Полный переход на SQLite + FastAPI + Dual-Mode

### Added (Server)
- **Полный переход на SQLite:**
  - Вся история, пользователи, группы, файлы теперь хранятся в SQLite
  - Удалено устаревшее JSON-хранилище (storage_legacy.py)
  - Повышение производительности и надёжности

- **FastAPI + WebSocket (Dual-Mode):**
  - REST API на порту 8000 для Web-клиентов
  - WebSocket endpoint `/ws` для обмена сообщениями в реальном времени
  - JWT-подобная аутентификация (без внешних библиотек)
  - Интерактивная документация Swagger: `/docs`

- **Консольное меню сервера:**
  - 4 главные опции (просмотр логов, администрирование, информация, выход)
  - ASCII-рамки с автовыравниванием
  - Полноценная панель администратора с 10 командами

- **Логгирование в файлы:**
  - `logs/chat.log` — история сообщений чата
  - `logs/error.log` — ошибки сервера
  - `logs/access.log` — доступы и действия администраторов
  - `logs/debug.log` — отладочная информация
  - `logs/events.json` — события в JSON формате

- **Стресс-тестирование:**
  - Тест на 450 одновременных подключений
  - Автоматическая отправка сообщений с заданным интервалом
  - Мониторинг производительности под нагрузкой

### Added (Client)
- **Автоматизированное тестирование:**
  - Тест-сьюит для Desktop клиента (10 проверок)
  - Проверка регистрации, авторизации, сообщений, файлов, групп
  - Автоматический запуск и закрытие GUI

### Changed (Server Architecture)
- **Dual-Mode работа:**
  - TCP сокеты (порт 5555) для обратной совместимости с Desktop клиентом
  - FastAPI (порт 8000) для Web-клиентов
  - Единое SQLite хранилище для обоих режимов

- **Безопасность:**
  - Собственная реализация JWT-токенов
  - Хеширование паролей с солью
  - Rate limiting для всех операций

### Fixed
- Исправлена ошибка `no such column: u.nickname` в групповых сообщениях
- Исправлена авторизация через сокеты (совместимость с новым хешированием)
- Исправлен эндпоинт `/api/users/online` (теперь не требует авторизации)
- Исправлены все проблемные импорты
- Устранены падения сервера при массовых подключениях

### Security
- Все логи сохраняются в отдельную папку
- Пароли не выводятся в консоль
- JWT токены защищены подписью

### Performance
- SQLite выдерживает 450 одновременных подключений
- Скорость обработки: ~20 регистраций/секунду
- Пропускная способность: ~150 сообщений/секунду под нагрузкой

### Known Issues
- Web-клиент требует разработки интерфейса (планируется на версию 1.3.0)
- Десктоп-клиент на PyQt6 в разработке
